PaaSを利用したWebアプリの開発・改修（2020）

業務用スーパーマーケットチェーンを展開されるお客様において、PaaS（サイボウズ株式会社「kintone」）上で動作するWebアプリの開発・改修を担当させていただきました。

使用技術

  • HTML / CSS / JavaScript
  • jQuery
  • kintone

業務内容

当案件は私を含めて2名という少人数体制で進めました。私が実装や試験といったコーディング部分、もう1人が工程管理やお客様との直接的な交渉などを担当しました。

kintoneを利用してアプリを新規開発したほか、お客様のほうで既にkintoneを用いていくつかの業務アプリを内製されていたため、それら過去に作られたアプリの改修や機能追加も行いました。

発揮したバリュー

新規開発・フロントエンド業務・kintoneの操作。そのいずれもが初ての経験でしたが、趣味でブログサイトを構築するなどフロントエンド技術には慣れていたこともあり素早くキャッチアップできました。

また、私が所属していた部署にはkintone経験者がおらず事実上1人で技術的課題を解決する必要がありましたが、技術への関心の強さを発揮して問題の洗い出しから解決までを1人で遂行することができました。

基幹業務システムの保守・改修（2019 - 2020）

業務用スーパーマーケットチェーンを展開されているお客様において、同社様に導入されている基幹業務システムの保守・改修を担当させていただきました。

使用技術

  • Visual Basic
  • SQL Server
  • JP1

業務内容

主たる業務は株式会社日立製作所のシステム運用管理ミドルウェア「JP1」を使用しての基幹業務システムの監視作業でした。システムが障害で停止しないように過去の稼働記録等を参考にして随時確認するとともに、障害が発生してしまったときは原因を調査してエスカレーションするか、あるいは監視チームの判断で復旧させました。

また、障害対応マニュアル、現行システム仕様書、1ヶ月単位でのシステム稼働記録レポートといった資料を作ったり、障害の再発を防止するストアドプロシージャを新たにシステムに追加したりといったコーディング業務を行いました。

発揮したバリュー

初めての案件、それもこれまでに意識したことがない保守という業務に戸惑うところは多々ありました。

ただ、監視業務の合間に現行システムのソースコードを読んで処理の流れや全体の動きを自分なりの図やメモにまとめたり、過去に起きた障害記録を読みかえすことでどのような箇所で障害が起きやすいかの傾向を把握するなどして既存の情報を自分なりに理解に落としこんでいくことでシステムの理解に努めました。

また、上長からプログラムを実装する能力に関して褒められており、お客様に提出するレポート作成を補助するツールを自作するなどして業務の効率化・定型化を進めました。

Work

Advanced Command-prompt Transpiler

独自言語で書かれたソースコードをCommand Promptで動作する形式のソースコードに変換するトランスパイラです。

Canvas APIとWeb Animations APIを使用したアニメーション実現処理を学ぶこと、また、Jestに代表されるテスティングフレームワークの操作に慣れるためにテストしやすい成果物になるように制作しました。

使用技術

  • HTML / Sass / TypeScript
  • Jest

制作期間

全体としてはおよそ1ヶ月かかりました。

アニメーションやトランスパイラといった処理部分は1週間から2週間ほどで組むことができましたが、どのようなコンテンツを作るのかという要件定義に半分以上の時間を割きました。

得られた知見

Canvas APIとWeb Animations APIの基本的な使用方法を習得しました。自身が思い描いたアニメーション表現を実装することができるようになりました。

テストコード・自動テスト・TDDといったテスト関係の技術の扱いに慣れました。どのようにテストを組み、どのようにテストを実施するかということについて基本的な流れを理解しました。

TOM Editor

シンプルなエディターライブラリです。

HTML・CSS・JavaScriptについて自身がどれくらい理解できているかを測るための試金石として制作しました。

使用技術

  • HTML / Sass / TypeScript

制作期間

構想から基本的なかたちになるまでに2週間ほどかかりました。うち半分近くはどのようなエディターを作るべきなのかという構想に消費したため実装に要したのは1週間ほどです。

得られた知見

エディターづくりを通して動的なDOMの書きかえやスタイルの操作によって複雑なWebページを構築するための経験を積みました。HTML要素を想定したとおりの位置に置き、ユーザーの操作に応じて思ったとおりの挙動をとるように実装することができるようになりました。

当初予想していたよりもプログラムの規模が膨らんだことで、プログラム全体のモジュール分けやMVCといったアーキテクチャにも関心を持ちました。最終的にはFluxを参考にして、エディターを構成する各モジュールを疎結合のまま組むことができるようになりました。

使用技術

  • Java

制作期間

プログラミング言語ごとにバラつきはあるものの、いずれも開発環境の構築から完成までに数日から1週間ほどかかりました。

得られた知見

さまざまなパラダイムに即したプログラムの構築経験を得ました。古いものから新しいもの、手続き型から宣言型、オブジェクト指向言語に純粋関数言語、と多種多様なプログラミング言語でゲームを作ったことで一通り全てのパラダイムでの実装経験を獲得しました。

また、これまでにあまり経験したことがなかった開発環境の構築という作業工程にも慣れることができました。
